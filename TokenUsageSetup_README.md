# SeaGrant Consortium – Monthly Usage & Billing Guide

## Overview

This document explains, in simple terms, how the system’s monthly costs work and how to keep them controlled and predictable.

Your application uses an AI service that charges based on how much it is used (similar to a utility bill like electricity or water). The good news is that your expected usage is very low and can be safely managed.

---

## Monthly Cost Plan ($22 Option)

We recommend setting a **$22 per month usage budget**. This is a safe, predictable amount that comfortably covers:

* One full monthly summary report
* Ongoing keyword searches
* Normal usage with room for growth

### What this means

* If usage stays normal → cost will likely be much lower (often under $5)
* The $22 acts as a **maximum cap**, not a fixed charge
* You only pay for what you use

---

## How Billing Works (Simple Explanation)

Think of it like a meter:

* The system counts how much text it processes
* Each request adds a very small cost (fractions of a cent)
* At the end of the month, you are billed for total usage

---

## Setting Up Billing Controls (Step-by-Step)

To ensure costs never exceed your budget, we strongly recommend the following setup:

### 1. Add a Payment Method

* Add a credit card to the billing account
* This allows the system to operate without interruption

---

### 2. Set a Monthly Spending Limit (IMPORTANT)

Set a **hard monthly limit of $22**

This ensures:

* The system will **never exceed this amount**
* If the limit is reached, usage simply pauses

---

### 3. Enable Usage Alerts

Set up automatic email alerts at:

* $5 → early notice
* $10 → mid-usage check
* $20 → near limit warning

This gives full visibility into spending throughout the month.

---

### 4. Internal Safeguards (Handled in the App)

The system itself is designed to be efficient:

* It minimizes unnecessary processing
* It avoids repeating work
* It limits the size of requests when possible

This keeps costs consistently low.

---

## What to Expect in Practice

For your specific use case:

* Typical monthly cost: **$1 – $5**
* Moderate usage: **$5 – $10**
* Heavy usage: **up to $22 (capped)**

---

## Why This Approach Works

This setup gives you:

* **Predictability** → Costs will not exceed $22
* **Transparency** → You see usage as it happens
* **Flexibility** → You only pay for what you actually use
* **Safety** → Automatic limits prevent unexpected charges

---

## Summary

* The system uses a **pay-as-you-go model**
* You set a **$22 monthly cap**
* Alerts keep you informed
* Actual costs will usually be much lower

This approach ensures the system remains both **cost-effective and fully controlled** for the SeaGrant Consortium.
